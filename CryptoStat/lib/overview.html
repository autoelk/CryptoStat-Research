CryptoStat is a Java program and class library for analyzing the randomness of cryptographic functions such as block ciphers, stream ciphers, authenticated ciphers, hash functions, and message authentication codes (MACs). A cryptographic function is supposed to perform a random mapping from its inputs to its outputs. CryptoStat samples a cryptographic function's outputs for various inputs and performs odds ratio tests for the hypotheses that the outputs are random or are nonrandom, for various definitions of randomness.

CryptoStat's main programs, classes Analyze and AnalyzeSweep, are parallel programs designed to work on a CUDA-capable GPU accelerated node.

The CryptoStat Library was developed by Professor Alan Kaminsky in the Department of Computer Science at the Rochester Institute of Technology. Professor Kaminsky's students Darshan Gandhi, Himanshu Kale, Vinayak Sasikumar, William Smith, and Jessica Sorrell contributed to earlier versions of the CryptoStat Library.

System Requirements

The CryptoStat Library requires the following software packages:

  •  Parallel Java 2 Library version 22-Feb-2018 or later — http://www.cs.rit.edu/~ark/pj2.shtml
  •  Nvidia CUDA Toolkit version 5.5 or later

Installation

First install the above software packages and set all requisite environment variables (such as PATH, CLASSPATH, LD_LIBRARY_PATH) as explained in the documentation for the above software packages.

There are two versions of the CryptoStat Library distribution, an executable distribution and a source distribution.

Executable distribution. The executable distribution comes in a Java Archive (JAR) file named "cryptostat_YYYYMMDD.jar", where YYYYMMDD are the year, month, and date. The executable distribution includes the CryptoStat Library class files only. To install the executable distribution, simply store the JAR file somewhere. You may wish to change the file name, say to "cryptostat.jar".

The executable distribution JAR file includes compiled PTX files for the GPU kernel functions. You might need to recompile these on your system. If so, install the source distribution (see below), recompile the PTX files, and replace them in the JAR file.

Source distribution. The source distribution comes in a Java Archive (JAR) file named "cryptostatsrc_YYYYMMDD.jar", where YYYYMMDD are the year, month, and date. The source distribution includes the CryptoStat Library class files, source files, and documentation files (Javadoc). To install the source distribution, just unpack the JAR file. The source distribution expands into a subdirectory named "CryptoStat". If you already have a subdirectory of that name which you want to save, be sure to unpack the JAR file in some other directory. Documentation is stored under the directory "CryptoStat/doc"; point your HTML browser to that directory. Java source files and class files are stored under the directory "CryptoStat/lib".

The source distribution includes CUDA source files for the GPU kernel functions as well as a script to compile them into PTX files. You might need to recompile these on your system. You might need to alter the compile script for your system.

Usage

Executable distribution. When compiling and executing Java programs that use the CryptoStat Library, if you have installed the executable distribution, you must set your classpath to include the CryptoStat JAR file and the Parallel Java 2 JAR file. Here is an example of a command for the bash shell to set the classpath to the current directory plus the JAR files (of course, use the proper path names for your system): 

export CLASSPATH=.:/home/fac/ark/public_html/cryptostat.jar:/home/fac/ark/public_html/pj2.jar

Here is an example of a command for the csh shell to set the classpath to the current directory plus the JAR files: 

setenv CLASSPATH .:/home/fac/ark/public_html/cryptostat.jar:/home/fac/ark/public_html/pj2.jar

You must also set your Java load library path as described in the Javadoc for package edu.rit.gpu in the Parallel Java 2 Library.

Source distribution. When compiling and executing Java programs that use the CryptoStat Library, if you have installed the source distribution, you must set your classpath to include the top-level directory under which CryptoStat's Java class files are stored as well as the Parallel Java JAR file. You must also set your Java load library path as described in the Javadoc for package edu.rit.gpu in the Parallel Java 2 Library.

Running CryptoStat. To analyze a cryptographic function with CryptoStat, run the Analyze program on a GPU-accelerated CUDA-capable parallel computer, specifying the cryptographic function to be analyzed, the input generator object for a series of A inputs, the input generator object for a series of B inputs, the odds ratio test object for the desired randomness hypothesis, and the bit group object for the output bit groups to be tested. Repeat for various choices of generator objects, test objects, and bit group objects.

The CryptoStat Library implements these cryptographic functions:
  •  AES-128 block cipher
  •  AES-192 block cipher
  •  AES-256 block cipher
  •  SHA-1 hash function
  •  SHA-224 hash function
  •  SHA-256 hash function
  •  SHA-384 hash function
  •  SHA-512 hash function
  •  SHA-512/224 hash function
  •  SHA-512/256 hash function
  •  SHA3-224 hash function
  •  SHA3-256 hash function
  •  SHA3-384 hash function
  •  SHA3-512 hash function
  •  SHAKE-128 extensible output hash function
  •  SHAKE-256 extensible output hash function

The CryptoStat Library provides these input generator objects:
  •  Gray code inputs
  •  One-off inputs
  •  Random inputs
  •  Sequential inputs
  •  Single input

The CryptoStat Library provides these input generator objects that perform transformations on input values:
  •  Add a constant value
  •  Complement all bits
  •  Permute selected bits
  •  Left-shift the bits
  •  Exclusive-or a constant value

The CryptoStat Library provides these input generator objects that combine two or more input value series together:
  •  Concatenated input series
  •  Interleaved input series
  •  Pasted-together input series

The CryptoStat Library provides these odds ratio test objects:
  •  Avalanche
  •  Difference
  •  Direct

The CryptoStat Library provides these bit group objects:
  •  Groups of adjacent bits
  •  Groups of nonadjacent (scattered) bits

Extending CryptoStat. You can add your own cryptographic function by writing a class that implements interface Function, along with the appropriate GPU kernel module. For an example, see source files AES128.java and AES128.cu.

You can add your own input generator object by writing a class that extends class Generator or a subclass thereof. For an example, see source file Seq.java.

You can add your own odds ratio test object by writing a class that extends class Test, along with the appropriate GPU kernel module. For an example, see source files Avalanche.java and Avalanche.cu.

You can add your own bit group object by writing a class that extends class BitGroup. For an example, see source file Adjacent.java.

Revision History

29-Apr-2019
Added package edu.rit.sha3, authored by Aziel Shaw, with implementations of the SHA-3 hash functions and the SHAKE extensible output hash functions.

15-Oct-2018
Moved all classes and interfaces, except the main program classes, into packages edu.rit.crst, edu.rit.aes, edu.rit.sha1, edu.rit.sha2, and edu.rit.util.

22-Feb-2018
Major change in the odds ratio tests performed. CryptoStat nows does a "run test" and a "birthday test" on each series of output bit group values. The run test looks at patterns of increasing or decreasing values. The birthday test looks at whether a block of consecutive values contains a collision (a repeated value).

30-Nov-2017
Changed the Analyze and AnalyzeSweep programs to include the current working directory and the Java class path in the provenance printout.

29-Nov-2017
Changed the AnalyzeSweep program optionally to restart a series of analysis runs in the middle.

26-Oct-2017
Changed the Analyze program to omit certain portions of the analysis if there is only one A input value or only one B input value. Also changed it to print the overall log Bayes factors for each round, aggregated over all A and B inputs and all bit groups; the number of nonrandom rounds is determined from these overall log Bayes factors. Made the same changes to the AnalyzeSweep program.

In input generator class OneOff: Added the OneOff(N) constructor.

Added input generator classes Paste, Rand, and Single.

In utility class BigInt: Added the randomize(prng,B) method.

09-Oct-2017
Another extensive rewrite of the CryptoStat Library.
